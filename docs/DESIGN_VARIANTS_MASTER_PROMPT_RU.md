# МАСТЕР-ПРОМПТ: премиальные варианты дизайна AiMA VPN

Ты — ведущий product designer, art director, motion designer и senior frontend engineer проекта **AiMA VPN / «Свободный интернет»**.

Твоя задача — не сделать один случайный интерфейс, а создать **6 полноценных премиальных дизайн-концепций**, реализовать их как переключаемые темы/варианты в одном проекте, сравнить по строгим критериям и выбрать лучший.

## Главная цель

Создать визуально выдающийся коммерческий продукт уровня лучших SaaS, fintech, AI и developer-platform интерфейсов 2026 года.

Интерфейс должен вызывать ощущения:

- надёжность;
- технологичность;
- свобода;
- скорость;
- премиальность;
- полный контроль;
- простота для обычного пользователя.

Не копируй конкретные сайты один в один. Извлекай лучшие принципы и формируй оригинальную систему AiMA VPN.

## Обязательные варианты

### Вариант 1 — Aurora Glass Prime

Основной кандидат.

- dark-first;
- глубокий графитовый фон;
- мягкие aurora-gradient слои;
- стекло используется умеренно и только для иерархии;
- тонкие светлые границы 1px;
- cyan, electric blue и violet как акценты;
- bento grid;
- мягкое рассеянное свечение;
- живой фон без дешёвого неона;
- премиальная типографика;
- плавные микровзаимодействия;
- состояние подключения визуализируется как энергетическое ядро.

### Вариант 2 — Liquid Metal OS

- чёрный, титан, хром, холодное серебро;
- жидкие металлические поверхности;
- эффект мягкого отражения;
- минимальная палитра;
- крупная кинетическая типографика;
- morphing-кнопки;
- панель похожа на интерфейс дорогой операционной системы будущего;
- никаких тяжёлых текстур и дешёвого 3D.

### Вариант 3 — Cyber Lux Network

- dark cyber-luxury;
- сетка, тонкие световые линии, узлы и маршруты;
- интерактивная карта серверов;
- cyan + magenta + royal blue;
- модульные карточки;
- реальные live-status индикаторы;
- визуальный акцент на скорости, маршруте и отказоустойчивости;
- эффект командного центра, но без перегруженности.

### Вариант 4 — Calm Enterprise

- уровень доверия Linear, Stripe, Vercel и Notion;
- почти без декоративных эффектов;
- точная типографика;
- много воздуха;
- тонкие границы;
- спокойные нейтральные цвета;
- один яркий акцент;
- интерфейс должен выглядеть дорого за счёт порядка, а не эффектов;
- приоритет: ясность, скорость, enterprise-доверие.

### Вариант 5 — Spatial Orbit

- 3D-пространство используется функционально;
- серверы представлены как орбитальные узлы;
- соединение пользователя визуально проходит через сеть;
- мягкий parallax;
- глубина слоёв;
- интерактивная география;
- WebGL включается только на мощных устройствах;
- обязательный 2D fallback.

### Вариант 6 — Neo Editorial Tech

- смелая editorial-композиция;
- крупные заголовки;
- асимметричные сетки;
- сочетание моноширинного и современного grotesk-шрифта;
- контраст белого, чёрного и одного яркого цвета;
- минимум стекла;
- максимум характера;
- подходит для лендинга и маркетинговых экранов.

## Какие экраны сделать для каждого варианта

1. Landing page.
2. Login / registration.
3. Главный dashboard пользователя.
4. Экран подключения VPN.
5. Карта серверов.
6. Подписка и тариф.
7. Устройства.
8. Telegram Mini App.
9. Admin dashboard.
10. Mobile Android layout.
11. Mobile iOS layout.
12. Empty, loading, error, offline и maintenance states.

## Главный пользовательский экран

На первом экране должны быть:

- большая кнопка «Подключить»;
- текущий статус;
- выбранный сервер;
- ping;
- скорость;
- активный протокол;
- оставшийся срок подписки;
- количество устройств;
- кнопка автоматического выбора лучшего маршрута;
- быстрый доступ к Telegram, WhatsApp, YouTube и Instagram;
- понятный индикатор Node 1 / Node 2;
- отсутствие технического мусора для обычного пользователя.

## Эффекты

Используй только эффекты, которые повышают качество:

- aurora gradient mesh;
- subtle glass blur;
- 1px luminous borders;
- animated noise texture;
- cursor proximity glow;
- card tilt с минимальной амплитудой;
- spring transitions;
- animated connection pulse;
- route drawing animation;
- number counters;
- skeleton loading;
- smooth theme transitions;
- tactile button press;
- restrained particles only in hero or success state;
- scroll-triggered reveal;
- soft parallax;
- controlled 3D depth;
- reduced-motion fallback.

Не использовать:

- тяжёлое размытие на каждой карточке;
- бесконечные вращающиеся элементы;
- кислотный неон на всём экране;
- перегруженные частицы;
- случайные 3D-объекты;
- дешёвые AI-иллюстрации;
- огромные бессмысленные hero-блоки;
- анимации, мешающие нажатию кнопок;
- декоративные эффекты без функции.

## Design system

Для каждого варианта создай:

- color tokens;
- typography scale;
- spacing scale;
- radius scale;
- shadows;
- blur levels;
- border styles;
- grid system;
- motion durations;
- easing curves;
- icon language;
- chart style;
- status colors;
- component states;
- mobile adaptations.

Все токены должны храниться централизованно через CSS variables и TypeScript theme definitions.

## Технологии

- Next.js App Router;
- TypeScript strict;
- Tailwind CSS;
- shadcn/ui или собственные headless-компоненты;
- Framer Motion/Motion для React;
- React Three Fiber только для Spatial Orbit;
- Lucide icons;
- next/font;
- next/image;
- CSS variables;
- Storybook;
- Chromatic или screenshot regression;
- Playwright;
- axe accessibility checks.

Не добавляй библиотеку ради одного простого эффекта.

## Производительность

- основная панель должна оставаться быстрой на среднем Android;
- эффекты должны использовать transform и opacity;
- избегать layout thrashing;
- lazy-load тяжёлые 3D-модули;
- не загружать WebGL до запроса пользователя или подтверждения мощности устройства;
- использовать next/image;
- использовать next/font;
- исключить layout shift;
- предусмотреть prefers-reduced-motion;
- обеспечить fallback без backdrop-filter;
- не ухудшать работу VPN-функций ради дизайна.

Целевые показатели:

- Lighthouse Performance >= 90 на desktop;
- Lighthouse Accessibility >= 95;
- Lighthouse Best Practices >= 95;
- CLS < 0.1;
- LCP < 2.5 s на разумном production-соединении;
- 60 FPS для основных анимаций на поддерживаемых устройствах.

## Сравнение вариантов

После реализации оцени каждый вариант по шкале 0–10:

1. Премиальность.
2. Оригинальность.
3. Доверие.
4. Простота.
5. Читаемость.
6. Скорость работы.
7. Mobile UX.
8. Accessibility.
9. Масштабируемость design system.
10. Соответствие бренду AiMA VPN.
11. Конверсия landing page.
12. Удобство ежедневного использования.

Создай таблицу `docs/design/DESIGN_COMPARISON.md`.

Не выбирай победителя только по красоте. Итоговый победитель должен иметь лучший общий баланс.

## Рекомендуемый итог

Ожидаемый кандидат на победу:

**Aurora Glass Prime + Calm Enterprise foundation + Spatial Orbit только в карте серверов.**

То есть:

- структура и читаемость Calm Enterprise;
- атмосфера и премиальность Aurora Glass;
- пространственный эффект Spatial Orbit только там, где он реально объясняет сеть;
- Cyber Lux элементы только для live network status;
- Liquid Metal используется дозированно в главной кнопке подключения и premium-акцентах;
- Neo Editorial Tech применяется на landing page.

Не смешивай всё сразу. Создай контролируемую гибридную систему.

## Рабочий алгоритм

1. Проведи аудит текущего UI.
2. Сохрани существующий рабочий дизайн.
3. Извлеки текущие цвета, шрифты, размеры и компоненты.
4. Создай wireframes.
5. Создай design tokens.
6. Реализуй 6 вариантов.
7. Сделай responsive-проверки.
8. Сделай accessibility-проверки.
9. Сделай performance-проверки.
10. Сравни варианты.
11. Выбери победителя.
12. Создай финальный гибрид.
13. Проверь все пользовательские функции.
14. Не объявляй готовность, пока preview routes, screenshots и тесты не подтверждены.

## Абсолютное правило

Не выдавай generic AI dashboard.

Каждая композиция должна иметь сознательную визуальную идею, собственную систему, понятную иерархию и обоснованные эффекты.

Если элемент выглядит красиво, но ухудшает скорость, читаемость или управление — упрости или удали его.
